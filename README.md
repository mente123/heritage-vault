# Heritage Vault — African objects at the British Museum

A map of **all 54 African countries** showing objects from the British Museum’s collection (search: [place=Africa](https://www.britishmuseum.org/collection/search?place=Africa) — **268,000+** (currently just 25,353 artifacts). Data is grouped **by country**; the left panel lists countries with object counts and a photo gallery with **year taken/acquired**.

## Run locally

The app loads **`data/africa_by_country.json`**. Serve over HTTP (not `file://`):

```bash
npx serve
```

Then open **http://localhost:3000** (or the port shown).

## Data: step3.csv and by-country build

- **`data/step3.csv`** — British Museum collection data (object_type, Museum number, findspot/production country and continent, acquisition_date, etc.). Used as the main **fact** dataset for African artefacts.
- **`data/step3_africa_by_country.json`** — Africa-only subset of step3: one entry per African country with `count` and `objects[]`. Generated by:
  ```bash
  node scripts/parse_step3_africa.js
  ```
  This keeps only rows where **findspot_continent** or **production_continent** is Africa, or findspot/production country is an African country. ~25,000+ artefacts across 49+ countries.
- **`data/africa_by_country.json`** — Main app dataset: 54 countries with `count`, `coordinates`, `objects[]`. Built from step3 Africa + contested objects + all-countries seed. Rebuild with:
  ```bash
  node scripts/build_africa_by_country.js
  ```
- **`data/africa_countries_centroids.json`** — List of 54 African countries with centroid coordinates for the map.
- **`data/british-museum-african-contested-objects.json`** — Contested/highlighted objects (Ethiopia, Nigeria, Ghana, Egypt, etc.) used to seed the by-country dataset.

### Getting more data from the British Museum (268k)

The BM site is behind Cloudflare and does not offer a direct bulk download of all 268k Africa results. You can:

1. **Scraper (Playwright)** — Run the script; it opens the BM search and extracts objects + production place, then maps place to country. First run may need a real browser to pass Cloudflare:
   ```bash
   npx playwright install chromium
   HEADED=1 node scripts/scrape_bm_africa.js   # optional: solve captcha in browser
   node scripts/scrape_bm_africa.js            # then run headless
   ```
   Output: `data/africa_scraped.json`. Then run `node scripts/build_africa_by_country.js` to merge into `africa_by_country.json`.

2. **CSV import** — If you obtain a CSV export (e.g. from [Museum Data Service](https://museumdata.uk/object-search/) or BM “Download results” when available), parse it and merge by country:
   ```bash
   node scripts/parse_bm_csv.js path/to/your_bm_africa_export.csv
   node scripts/build_africa_by_country.js
   ```
   The parser looks for columns like “Production place”, “Object name”, “Museum number”, “Object URL” and maps place text to country.

## Map and UI

- **Map** — One marker per country (54 countries). Marker size reflects object count in the dataset. Click a country to see its objects and highlight the region. Clustering when zoomed out.
- **Left panel** — “Objects” gallery (photo + year); “By country” list with count per country; click a country to see object chips and open detail (history, BM link).
